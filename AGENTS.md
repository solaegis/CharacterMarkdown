## Learned User Preferences

- When asked to implement an attached `*_plan.md` build plan, do not edit the plan file; update the character profile export, alignment checklists, and related artifacts only.
- Do not recreate todos when the user says todos already exist; mark them in progress and complete them in order.
- Example build plans should use current in-game ESO names (e.g. Champion Point stars), not deprecated pre–Update 29 labels.
- User expects ESO mechanical correctness in example build documentation: skill bars must match equipped weapon type, each morph appears at most once, and bar tables include ultimates; document **slotted morph names as shown in the skills UI** (not unmorphed base or wiki-only labels—e.g. **Inspired Scholarship → Recuperative Treatise**, not **Tome-Bearer's Inspiration** after morphing).
- ESO **companion gear is separate from player character gear** (`Companion's` items, companion-only traits and crafting stations); companion loadout advice must not reuse player set pieces from **Gear and crafting** (e.g. Julianos/Divines on the character).
- Target gear in build plans must always be **craftable** (account crafter / crafting stations). Do not plan primary loadouts around overland farming, dungeon monster sets, trial drops, or other non-crafted acquisition unless the user explicitly asks for an exception.
- When changing addon settings (defaults in `src/settings/Defaults.lua`, LAM panel labels/checkboxes/presets in `src/settings/Panel.lua`, or `ToggleAllSections`), always update `docs/settings-presets-matrix.md` in the same change so factory defaults and every preset column stay accurate.
- Do not bulk-reformat existing `examples/solaegis/na/*_plan.md` files to the canonical structure unless the user explicitly asks; apply the exemplar outline to new and heavily revised plans only.

## Learned Workspace Facts

- Character examples live under `examples/{account}/{location}/{slug}.md` (identity = account + location + slug; same slug allowed across different accounts or locations, not duplicated for the same account and location). Run `task examples:check` after adding or removing profiles.
- Solaegis NA character examples and build plans live under `examples/solaegis/na/` (paired profile markdown + `{slug}_plan.md`). Non-character docs live in `examples/fixtures/`.
- Solaegis-style build plans use the canonical section order in `examples/solaegis/na/silent_snow_falls_plan.md` (structure exemplar). Reference `docs/subclassing.md` for Trinity / subclassing theory. Paired `{slug}_plan.md` + character profile live under `examples/{account}/{location}/`.
- **Build plan H2 order (required for new and heavily revised plans):** (1) title + profile link + pitch, (2) **Build at a glance** (stats, mundus, sets, bars, companion, primary mount + doc-map links), (3) **Roleplay**, (4) **Trinity configuration** (mermaid + one unified table), (5) **Combat kit** (`### Skill bars`, `### Rotation`, `### Passive skills`), (6) **Gear and crafting** (`### Set rationale`, `### Target loadout`, `### Crafting handoff`), (7) **Champion Point Mapping**, (8) **Companion strategy**, (9) **Collectibles** (`### Mount`, `### Pet`, `### Dye and style`), (10) **Next steps / in-game checklist** with gear phases as labeled steps — no separate Gear Phases H2.
- **Build plan structural rules:** one execution checklist only (merge gear phases into it); no duplicate H2 emoji/titles; trinity = mermaid + one table (no third slot-only table); mount and visual identity under Collectibles, never between Gear and Champion Points; **Roleplay** includes **Suggested Custom Title** (≤100 chars) and **Build Notes** paste text (≤1,900 chars) for LAM Custom Title / Build Notes; optional Tier B polish when creating plans — use Gear and Crafting subsections and passives immediately after skill bars within Combat kit; skeleton template at `examples/templates/build_plan_template.md`.
- Crafting support specs for builds may be separate markdown files (e.g. `examples/fixtures/karakedi_crafting.md`) referenced from the main plan.
- `examples/solaegis/na/masisi.md` documents the account crafter used for gear-crafting plans tied to other Solaegis characters.
- CharacterMarkdown exports in `examples/` are source of truth for in-game state; when syncing a plan from a profile, label **live** (equipped) vs **target** in Build at a glance and gear, and keep skill bars aligned with slotted actives and equipped weapons from the export.
- Paired character profiles should link to their `{slug}_plan.md` with a section anchor table; highlight the plan checklist as the primary in-game entry point.
- `docs/settings-presets-matrix.md` is the canonical matrix of factory defaults vs LAM presets (Minimal, Solo PvE, PvP Build, Achievement Hunter, Crafter) for every settings-menu checkbox; keep it in sync whenever settings code changes.
- LAM presets call `ToggleAllSections(false)` first; any checkbox not turned back on by the preset stays off until the user uses Reset to Defaults or toggles it manually.
